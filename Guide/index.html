I. Tâches Annuelles

1. Remplacement des liens

  • Fichiers : index.htmlevenement.jsonactualites.json
  • Etapes pour index.html :
    1. Chercher la balise "a" contenant le lien qu'on souhaite remplacer <a href="https://exemple">le texte qui apparait sur la page</a>
    2. Remplacer l'attribut href par le nouveau lien, ainsi que le texte si nescessaire <a href="nouveau https://">Nouveau text</a>
  • Etapes pour les fichiers JSON :
    1. Chercher l'attribut "lien" contenant le lien qu'on souhaite modifier "lien": "https://exemple
    2. Remplacer le lien que l'attribut contient "lien": "nouveau https://

2. Mise à jour des tarifs

  • Fichier : index.html
  • Étapes :
    1. Rechercher tarifs-container
    2. Modifier les prix dans <div class="tarif-price">
    3. Mettre à jour les descriptions dans <ul class="tarif-features">

3. Changement des horaires

  • Fichier : index.html
  • Étapes :
    1. Localiser la section <section id="horaires">
    2. Modifier les créneaux dans les balises <p> sous chaque <h3>

4. Mise à jour des statistiques

  • Fichiers : script.js index.html
  • Étapes :
    1. repérer dans cette section dans script.js
      2. const CONFIG = { stats: { licencies: 125, jeunes: 86, animateurs: 6 } };
    2. Modifier le nombre de licencies, de jeune et d'animateur à la demande
    3. puis aller dans index.html
    4. localiser puis modifier egalement les statiqtique presentent dans <section id="horaires"><div class="stats-banner">

II. Tâches Hebdomadaires

1. Ajouter une actualité

actualites.JSON { "date": "AAAA-MM-JJ", "titre": "Titre de l'actualité", "description": "Description complète...", "image": "chemin/vers/image.jpg", "lien": "https://lien-externe.com" }

Tips :

  • Placer les nouvelles actualités en haut du fichier
  • Les images doivent être ajoutées dans images/Actu/
  • Les images ne doivent pas dépassé 100Ko, si la photo faits plus, utiliser un outils de compression en ligne comme https://squoosh.app/

2. Ajouter un événement à venir

evenements.JSON { "date": "AAAA-MM-JJ", "titre": "Nom de l'événement", "description": "Détails...", "image": "chemin/vers/image.jpg", "lien": "https://lien-inscription.com" }

Tips :

  • Placer les nouveaux événements en haut du fichier et supprimé les évenements passé
  • Les images doivent être ajoutées dans images/Event_Temporaire/
  • Les images ne doivent pas dépassé 100Ko, si la photo faits plus, utiliser un outils de compression en ligne comme https://squoosh.app/

3. Ajouter des archives photos

archives-photos.JSON { "year": "2025", "events": [ { "title": "Nom de l'événement", "date": "JJ Mois AAAA", "description": "Description...", "images": ["chemin/photo1.jpg", "chemin/photo2.jpg"] } ] }
  • Important :
  • Pour les photos, créer un nouveau dossier dans images/Tournoi_AAAA/Nom_Tournoi
  • Les images ne doivent pas dépassé 100Ko, si la photo faits plus, utiliser un outils de compression en ligne comme https://squoosh.app/
  • Utiliser le même nom de dossier dans le JSON

III. Ajouter une Nouvelle Année

Pour les archives photos :

  • Ouvrir archives-photos.json
  • Ajouter un nouveau bloc en haut : { "year": "2026", "events": [] }
    • Puis ajouter les événement entre les crochets

Pour le filtre d’année :

  • Fichiers : archives-actualites.htmlarchives-photos.html
  • Étapes : <button class="archive-year-btn" data-year="2026">2026</button>

IV. Bonnes Pratiques

  • Validation des liens :
    • Tester tous les liens après modification
    • Utiliser des URL complètes https://...
  • Gestion des images :
    • Taille max : 100Ko par image
    • Formats acceptés : JPG, PNG, JPEG
    • Nommer les fichiers sans espaces (ex: tournoi_noel_2025.jpg)
  • Sauvegardes :
    • Toujours faire une copie des fichiers avant modification
    • Utiliser des commentaires dans le JSON pour se repérer : // 2025 - Tournoi d'été { ... }

V. Dépannage Rapide

Problème Solution
Image non affichée Vérifier le chemin dans le JSON
Date incorrecte Respecter le format AAAA-MM-JJ
Bouton filtre manquant Vérifier l’attribut data-year
Texte mal formaté Éviter les caractères spéciaux

VI. Mettre en ligne le site

  • Outils : VSCODEGit & GitHub
  • Étapes :
    1. Sur VSCODE tester localement les ajouts/modification grace à LiveServeur (GoLive)
    2. Si tout est bons, cliquer sur la barre gauche de VSCODE, sur l'icone avec les liens : Source Control
    3. Ajouter un message expliquant les modifications faites puis sur le bouttons commit, appuyer sur la petite fleche et choisisser : Commit and Push
    4. Si tout est bien Configurer, au bout de quelques minutes, le site est censé etre mis à jour, tester en retapant l' URl du site, si ça ne marche pas, contacter moi ci-dessous en m'expliquant le probleme
